## **TSR**

В используемой реализации [TSInterpret](https://github.com/fzi-forschungszentrum-informatik/TSInterpret/tree/main) TSR используется не как отдельный метод получения атрибуций, а как **постобработка базовой saliency-карты**.
Сначала выбирается один из стандартных методов интерпретации из Captum (`GRAD`, `IG`, `DL`, `GS`, `SG`, `SVS`, `FA`, `FO`), после чего, к его атрибуциям применяется процедура **Two-Step Rescaling**.

Идея TSR состоит в двухэтапной декомпозиции важности признаков:

1. сначала оценивается **вклад каждого момента времени**;
2. затем для наиболее значимых моментов времени оценивается **вклад отдельных признаков**.

В анализируемой постановке для TSR сначала вычисляется исходная attribution map `ActualGrad` для выбранного метода интерпретации. Затем для каждого временного шага `t` формируется модифицированный вход, в котором все признаки в момент времени `t` заменяются значением `assignment`. Для модифицированного входа повторно вычисляется attribution map, после чего определяется суммарное абсолютное отклонение новой карты от `ActualGrad`. Это отклонение интерпретируется как вклад временного шага `t` и образует вектор `timeContribution`.

Далее, если временной ряд многомерный, для временных шагов с высоким вкладом дополнительно оценивается вклад каждого признака. Для этого поочерёдно маскируется отдельный признак `c` в момент времени `t`, после чего снова измеряется изменение attribution map относительно `ActualGrad`. Так формируется `featureContribution`.

Итоговая карта TSR вычисляется как
$$\text{newGrad}[c,t] = \text{timeContribution}[t] \cdot \text{featureContribution}[c,t]$$
После этого берётся абсолютное значение карты и выполняется min-max нормализация.

Ниже — интуитивные объяснения выбранных методов, написанные так, чтобы их можно было использовать в отчёте или статье. Я постарался держать уровень между формальным и интуитивным, как обычно делают в секциях Methods / Background.

## **Интуиция градиентных методов объяснений**

##### **GRAD (Vanilla Saliency / Gradient)**

Метод Vanilla Gradient является базовым градиентным методом объяснений. Его идея состоит в вычислении градиента выхода модели по входным признакам:

$$S_i = \left|\frac{\partial f(x)}{\partial x_i}\right|$$

где
- $f(x)$ — предсказание модели для входа x,
- $x_i$ — i-й входной признак.

Интуитивно градиент показывает, насколько сильно изменится предсказание модели при небольшом изменении данного признака. Если значение градиента велико, значит даже небольшое изменение соответствующей временной точки существенно влияет на выход модели. Таким образом, такие точки считаются важными для принятого предсказания.

Однако этот метод часто приводит к шумным картам важности, поскольку градиенты нейронных сетей могут сильно колебаться даже при небольших изменениях входных данных.


##### **GS (Gradient SHAP)**

Метод Gradient SHAP объединяет идеи градиентных методов и значений Шепли. Он оценивает вклад каждого признака как ожидаемое значение градиентов между входом и случайными baseline-точками.

Практически это реализуется следующим образом:
	1.	выбирается набор случайных baseline-входов;
	2.	между baseline и реальным входом генерируются случайные точки;
	3.	в этих точках вычисляются градиенты;
	4.	результаты усредняются.

Интуитивно метод можно понимать так:
вместо того чтобы смотреть градиент только в одной точке, мы смотрим в среднем по множеству точек между baseline и входом.

Это уменьшает влияние локального шума и позволяет получить более стабильные оценки важности признаков.



##### **SG (SmoothGrad)**

Метод SmoothGrad направлен на снижение шума, характерного для обычных градиентных карт. Его основная идея состоит в том, чтобы усреднить градиенты по множеству зашумленных копий входного сигнала.

Процедура метода:
	1.	к исходному входу добавляется случайный шум;
	2.	для каждого зашумленного входа вычисляется градиент;
	3.	полученные карты важности усредняются.

Формально:

$$S = \frac{1}{N}\sum_{i=1}^{N} \left| \nabla_x f(x + \epsilon_i) \right|$$

где $\epsilon_i$ — случайный шум.

Интуитивно это похоже на сглаживание карты важности: случайные шумовые артефакты усредняются, а устойчивые паттерны остаются. В результате карта важности становится более интерпретируемой и менее чувствительной к локальным колебаниям градиентов.

##### **DL (DeepLIFT)**

Метод DeepLIFT (Deep Learning Important FeaTures) оценивает вклад каждого признака относительно некоторого baseline-входа. Вместо анализа локальных градиентов он рассматривает разницу между активациями сети для реального входа и baseline.

Для каждого нейрона вычисляется вклад:

$$C_{\Delta x_i} = \frac{\Delta y}{\Delta x_i}$$

где

$\Delta y = f(x) - f(x_0)$ и $x_0$ — baseline-вход.

Интуитивно DeepLIFT отвечает на вопрос:

насколько изменилось предсказание модели по сравнению с нейтральным входом и какие признаки внесли наибольший вклад в это изменение.

В отличие от обычных градиентов, DeepLIFT способен корректно распространять вклад через нелинейности и насыщенные участки сети, где градиенты могут быть близки к нулю.


##### **DLS (DeepLIFT SHAP)**

Метод DeepLIFT SHAP объединяет идеи DeepLIFT и вновь значений Шепли. Он использует несколько baseline-входов и усредняет вклады DeepLIFT относительно каждого из них.

Процедура метода:
	1.	выбирается набор baseline-входов;
	2.	для каждого baseline вычисляется карта вкладов DeepLIFT;
	3.	результаты усредняются.

Интуитивно метод можно рассматривать как аппроксимацию значений Шепли для признаков, где DeepLIFT используется для быстрого вычисления вкладов. Это делает оценки важности более устойчивыми к выбору baseline и лучше отражающими глобальную структуру данных.
