## **About**

## **Постановка задачи**

Задача классификации — стандартная задача машинного обучения. В случае временных рядов мы имеем вектор, размерности `[feats, time]`, где при `feats=1` задача сводится к обычной табличной классификации. В такой постановке для временных рядов анализ признаков становится сопряжен с анализом временных точек признаков.

В данной работе мы рассматриваем для анализа значимых временных точек градиентные методы объяснения. В общем смысле они строят **saliency map**, то есть карту важности точек входного временного ряда для предсказания модели. 

В частности, рассматривается задача **сравнения**:

- gradient-based explanations для временных рядов в прямом виде 
- gradient-based explanation с применением техники **Temporal Saliency Rescaling (TSR)**

## **Research Question**

**Делает ли TSR градиентные объяснения для временных рядов более информативными и более согласованными с поведением модели по сравнению с исходными saliency maps?**

## **Дизайн эксперимента**

На датасете одномерных временных рядов [FordA](https://www.timeseriesclassification.com/description.php?Dataset=FordA) обучается модель классификации, после чего для её предсказаний строятся explanations двумя способами:

1. обычные gradient saliency maps;
2. saliency maps после применения TSR.

Далее полученные объяснения сравниваются количественно, с помощью deletion/insertion test, оценивающих, насколько выделенные объяснением точки действительно влияют на предсказание модели.

#### **Данные**

В качестве тестовой задачи используется датасет **FordA**. Каждый объект представляет собой одномерный временной ряд длины 500. Обучающая выборка содержит 3601 объектов, тестовая используется для оценки модели и анализа explanations.

## **Ограничения**

**Исследование пилотно.** Авторы понимают, что результаты, полученные на одной модели и на конкретных данных не эксполируются на разные данные/другие модели. Основная цель проекта — построение baseline и тестрование "живости" поставленного RQ. Если результаты окажутся содержательными, эксперимент можно расширить на другие датасеты, архитектуры и методы интерпретации.
